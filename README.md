Hotel Reservation System Database

Project Overview

This project implements a Hotel Reservation System using MariaDB. The aim of the project is to design and develop a relational database capable of managing hotel operations, including customers, room types, rooms, bookings, payments, and additional hotel services.

The project was developed as part of the B103 Databases & Big Data module at Gisma University of Applied Sciences.

Database Features

Customer Management
Room Management
Booking Management
Payment Tracking
Service Management
Revenue Analysis

Database Tables

The database consists of the following tables:

1. Customers
2. RoomTypes
3. Rooms
4. Bookings
5. Payments
6. Services
7. BookingServices

The BookingServices table acts as a junction table to resolve the many-to-many relationship between bookings and services.
